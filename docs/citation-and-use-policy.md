# Citation and Use Policy
## Cinematic Emotion Lab

> This document defines how the Cinematic Emotion Lab repository should be cited
> in academic, professional, and educational contexts, and what constitutes
> acceptable use of the repository's public materials.

---

## Citation Requirements

### When to Cite

Cite this repository when you:

- Reference the research methodology, experimental design, or annotation framework
  described here in any published or distributed work
- Use the public corpus subset (acoustic feature vectors) in your own analysis
- Use or adapt the annotation schema (`datasets/annotations/schema.json`)
- Build on the Composer Gap conceptual framework
- Reproduce or extend the feature extraction pipeline from NB-EXP-001
- Reference any written research documentation from this repository in academic work

### Machine-Readable Citation

A machine-readable citation is provided in `CITATION.cff` at the repository root.
GitHub renders this automatically in the repository sidebar. Most citation management
tools (Zotero, Mendeley, etc.) can import CITATION.cff directly.

### Formatted Citations

**APA**
Bernard G. (2026). *Cinematic Emotion Lab* [Software and research documentation].
GitHub. https://github.com/bernardvgosh/cinematic-emotion-lab

**MLA**
G., Bernard. *Cinematic Emotion Lab*. 2026, github.com/bernardvgosh/cinematic-emotion-lab.

**BibTeX**
```bibtex
@software{bernardg2026cinematicemotionlab,
  author       = {G., Bernard},
  title        = {Cinematic Emotion Lab},
  year         = {2026},
  publisher    = {GitHub},
  url          = {https://github.com/bernardvgosh/cinematic-emotion-lab},
  note         = {Interdisciplinary AI research program: cinematic emotion
                  recognition, harmonic tension modeling, and computational
                  analysis of composer-machine interpretive divergence}
}
```

**Chicago**
G., Bernard. "Cinematic Emotion Lab." 2026. GitHub.
https://github.com/bernardvgosh/cinematic-emotion-lab.

### Citing the Annotation Schema Specifically

If your work specifically uses or references the annotation schema:

```bibtex
@misc{bernardg2026annotationschema,
  author    = {G., Bernard},
  title     = {Cinematic Emotion Annotation Schema v1.0},
  year      = {2026},
  url       = {https://github.com/bernardvgosh/cinematic-emotion-lab/blob/main/datasets/annotations/schema.json},
  note      = {Cinematic Emotion Lab. Composer-intent annotation framework
               for cinematic audio emotion recognition.}
}
```

### After Publication

When the formal peer-reviewed paper associated with this research is published,
please update your citations to reference that paper as the primary academic source.
This CITATION.cff file will be updated at that time with the preferred academic
citation. A notice will be committed to the repository.

## Acceptable Use

### Academic Research

Use of the public materials — code, feature extraction pipeline, annotation schema,
documentation, and public corpus subset — for academic research is permitted under
the terms of the repository LICENSE with attribution. This includes:

- Comparative studies using the public feature subset alongside other datasets
- Replication studies using the public pipeline on your own audio material
- Critical analysis of the methodology in academic publications
- Extension of the research framework in your own work, with citation
- Reference to the Composer Gap conceptual framework, with citation

### Educational Use

Use of repository materials in university courses, workshops, tutorials, and
educational programs is permitted with attribution. The repository may be referenced
as a case study in MIR methodology, affective computing, or interdisciplinary AI
research.

### Commercial Use

Commercial use of code is permitted under the MIT License (with attribution).
Commercial use of research documentation, schemas, visual assets, and written
content is not permitted without prior written permission from the author.

For commercial licensing inquiries: contactbernardg@gmail.com

### Derivative Research

Derivative research that builds substantially on the methodology, annotation
framework, or conceptual contributions of this repository should:

1. Cite this repository in any publication or presentation
2. Clearly distinguish your contributions from those of this research program
3. Not represent the methodology as your own original contribution without attribution
4. Contact the author if the derivative work represents a substantial extension that
   might benefit from coordination or collaboration

## Attribution Wording

When crediting this repository in non-citation contexts (blog posts, conference
presentations, LinkedIn, README files of derivative projects), the following
attribution wording is appropriate:

**Short form:**
> Cinematic Emotion Lab - Bernard G. (2026)
> https://github.com/bernardvgosh/cinematic-emotion-lab

**Long form:**
> This work builds on the Cinematic Emotion Lab research program by Bernard G. (2026),
> an interdisciplinary AI research initiative at the intersection of music information
> retrieval, affective computing, and computational musicology.
> https://github.com/bernardvgosh/cinematic-emotion-lab

## What This Repository Is Not

To avoid misrepresentation, note the following:

- This is not a published paper. Cite it as a software/repository, not as a journal
  article, until the formal publication is available.
- This is not a finished product. It is an active research program with staged
  public disclosure. Results and conclusions are not yet public.
- The public corpus subset is not the full research dataset. Do not describe it as
  such in your work. Clearly distinguish the public subset from the full corpus.
- The methodology documented here may be refined before final publication. If you
  reference specific methodological details, note the version and date of the
  repository you consulted.
